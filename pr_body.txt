## Summary

Establishes Phase 0: the Witness Foundation Shell. The app shell is now organized around a focused Witness identity instead of a broad multi-feature hub:

- primary navigation is now **Witness / Record / Settings**;
- `home` remains the implementation route for the Witness landing surface;
- `profile` remains the implementation route for Witness Record;
- Library/Experiences, Programs, and Achievements remain available as secondary destinations;
- Witness Home now emphasizes the next observation with “Observe what others miss” and “Begin Observation” copy;
- existing result content is wrapped in a structural `EvidenceRevealContainer` for future reveal work;
- reusable shell foundation components and asset placeholder folders were added.

## User Experience Impact

The app now feels more like an intentional observation instrument:

- the first primary action is to begin a Witness observation;
- Record is presented as the player’s personal observation history rather than a generic profile tab;
- secondary destinations are still accessible, but quieter;
- Home no longer foregrounds achievement/streak/program clutter as the main reason to open the app.

## Technical Impact

- Kept the existing `AppShell`, `NavigationService`, `AppRoutes`, and route history model.
- Kept internal route names stable where possible.
- Removed `experiences` from bottom-tab navigation only; the route and screen remain intact.
- Added lightweight reusable components:
  - `ScreenContainer.gd`
  - `ModalLayer.gd`
  - `EvidenceRevealContainer.gd`
- Added future asset pipeline placeholders under `app/assets/{scenes,evidence,home,record,branding}`.
- Updated Phase 0 documentation and store-preparation recommendations.
