# Google Play Store Evolution Plan

**Purpose:** evolve store presence in step with the real flagship experience.
**Rule:** store assets and claims must show the runnable product, not source artwork, trailer-only fiction, or unvalidated future systems.

---

# 1. Store positioning

## Core storefront promise

> **Catch the detail that mattered. See what you missed.**

Two Second Witness should be positioned as a premium offline observation experience:

- short fair Witness Moments;
- ordinary scenes that disappear before one question;
- evidence-first reveals;
- a private Witness Record;
- no account, ads, or remote gameplay dependency in current release configuration.

Avoid store positioning as brain training, diagnosis, IQ, cognitive assessment, generic memory test, campaign mystery, or “solve cases” game.

---

# 2. App icon plan

## Current suitability

The current icon/brand system uses an eye/witness motif with dark/purple/gold language. It is conceptually appropriate: it communicates attention, observation, and premium identity more directly than an object sprite or screenshot.

Current unknowns:

- real launcher readability at small Android sizes;
- coherence with latest grounded warm scene art;
- recognition against competing Play listings;
- whether the eye promises mystery/thriller beyond actual gameplay.

## Evolution direction

- Preserve the eye as the durable product symbol.
- Prefer one bold, readable witness signal over scene-detail clutter or text.
- Harmonize dark frame, violet identity, and warm evidence accent only after Premium Presentation Layer is validated.
- Do not change package identity or introduce a new icon solely because a later update needs marketing novelty.

## Timing

| Stage | Icon action |
|---|---|
| Current / Updates 1–3 | Validate existing icon on physical launchers; no speculative replacement. |
| Update 6 | If flagship art direction is proven, create/test refined icon variants. |
| Update 10 | Finalize icon only if device/store tests show a measurable identity/readability benefit. |

# 3. Feature graphic plan

## Visual message

The feature graphic should communicate one emotional product promise in one glance:

```text
Witness eye / attention frame
+ one readable ordinary scene detail
+ “Catch what mattered” emotional premise
```

It should not show five game cards, an oversized dashboard, a long feature list, small text, device frames, unsupported awards, or a cinematic plot scene absent from gameplay.

## Emotional promise

- An ordinary moment has hidden importance.
- The player will get a fair second look.
- This is calm, premium, and visually distinct.

## Update timing

- Use current graphic only if it honestly matches current runnable experience.
- Update after Updates 2, 3, and 6 have produced real scene/reveal/presentation captures.
- Lock final feature graphic at Update 10 after storefront/first-impression review.

# 4. Screenshot strategy

Screenshots should tell one coherent player story, not display every menu.

## Required screenshot sequence

| Order | Screenshot | What it proves | Capture rules |
|---:|---|---|---|
| 1 | **Witness Moment / Brief** | One current moment worth noticing; clear premium entry proposition. | Real device build; no mock UI; minimal readable copy. |
| 2 | **Observation** | Ordinary scene under brief attention, scene stage as hero. | Show actual timer/scene framing; no fabricated two-second claim if policy differs. |
| 3 | **Recall** | One clear factual witness call after scene concealment. | Large real response controls; no clutter. |
| 4 | **Evidence Reveal** | Signature payoff: scene context and highlighted proof. | Most important screenshot; use factual visible evidence. |
| 5 | **Witness Record** | Private continuity/history, not score dashboard. | Show archive/familiarity only after Update 5 validates it. |

## Optional supporting screenshots

- Scene-world variety: Office/Kitchen/Workshop/Travel/Garden.
- Brief completion/resume state.
- Accessibility settings showing control, not feature clutter.
- Spot the Difference as a companion variation only after flagship sequence is clear.

## Capture standards

- Capture from signed/internal Android build on physical hardware.
- Use supported portrait dimensions and clear status/safe-area behavior.
- Do not overlay marketing copy that hides gameplay evidence.
- Avoid source art mockups, editor-only captures, debug labels, prototype copy, or unsupported narrative imagery.
- Ensure high-text-size/color/contrast claims match actual settings behavior before depicting them.

# 5. Store description evolution

## Current product description — baseline version

**Short concept:**

> Study a scene, answer what you noticed, and see the evidence return.

**Long description direction:**

> Two Second Witness is a premium offline observation game built around short, fair Witness Moments. Look closely at an ordinary scene before it disappears, make one call about what mattered, and see the evidence revealed in context. Explore Scene Investigation, Flash Words, Spot the Difference, Object Recall, and Pattern Recall; build a private Witness Record with local progress, tutorials, and accessibility controls. No account or ads required.

Use only claims verified in the shipped build.

## Major-update description — flagship foundation version

After Updates 1–5 validate core behavior:

> Every visit begins with a moment worth noticing. Two Second Witness centers Scene Investigation: observe an ordinary scene, remember what mattered, make one witness call, and see exactly why the answer was true. A finite Witness Brief gives you a calm way to return, while your private Witness Record remembers the moments you have seen.

## Flagship release description — Update 10

> Catch the detail that mattered.
>
> Two Second Witness is a premium, private observation experience. Enter a Witness Brief, study an ordinary scene before it disappears, and make one honest call about what you saw. Then watch the evidence return: the object, relation, or count that was always there.
>
> Scene Investigation leads a growing world of fair, replayable moments, supported by companion observation challenges. Build a personal Witness Record of scenes, discoveries, and familiar places—without ads, accounts, or pressure.
>
> • Fair scene-based Witness Moments
> • Evidence-first reveals that explain every answer
> • Fresh finite Witness Briefs and safe local progress
> • Offline play with no account required
> • Accessibility controls for timing, text, contrast, motion, sound, and haptics

Do not add claims of cognitive improvement, real-world ability, Story Mode, connected Threads, or cloud behavior unless those are genuinely shipped and validated.

# 7. Existing-user reintroduction

Returning users should discover evolution through play, not a forced feature tour.

## Principles

- Do not block a returning player with a long “everything changed” overlay.
- Lead with the current Witness Brief and one concise, optional line about the relevant evolution.
- Let the transformed evidence reveal demonstrate Update 2 before explaining it.
- Let the Witness Record become visible after enough normal moments, not as an obligatory migration screen.
- Preserve all existing local progress and explain any visible terminology change calmly.
- Never imply that an existing user is behind, has lost a streak, or needs to restart.

## Example respectful reintroduction

> **A moment worth noticing is ready.**
>
> **The evidence now returns in the scene—take another look after your answer.**

Then enter normal play. The product itself should prove that it has become more than the user remembers.
